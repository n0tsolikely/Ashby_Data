1) REVIEW TIERS (HOW MUCH TO REVIEW EACH SESSION)
================================================================================
Tier 0 (Always Review — Mandatory before any Raid work)
- Continuity Lock + Bootstrap Prompt (if provided for the session)
- Active Guild Orders (current raid)
- Quest Board state (Accepted + Completed)
- Latest Control Sync Snapshot
- Latest End-of-Day Snapshot
- Confirm canonical engine repo exists and tests can run

Tier 1 (Review Only When Relevant to the quest being executed)
- Codex sections relevant to the subsystem touched (storage, truth, overlays, indexing, profiles, etc.)
- Prior quests that are direct dependencies of the current quest
- Canonical Vision docs ONLY if the quest changes UX/flows/contract wording

Tier 2 (Do NOT review unless troubleshooting a contradiction)
- Old snapshots (beyond latest CS + latest EOD)
- Idea dumps / To Do lists / long-term brainstorming
- Archives / deprecated contracts
