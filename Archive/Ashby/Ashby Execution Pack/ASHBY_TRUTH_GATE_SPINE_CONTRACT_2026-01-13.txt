Contract: Evidence model (evidence.py)
Truth gating cannot run on chat history.
It runs on structured evidence.

Define these canonical dataclasses/types:

1) Citation (optional but platform-wide useful)
Represents a pointer into a durable artifact.
Required fields:
- session_id: str
- artifact_path: str        # relative path under the artifact storage root
- segment_id: str | None
- start_ms: int | None
- end_ms: int | None
- speaker_id: str | None

Rules:
- start_ms/end_ms, if present, are integer milliseconds with start_ms <= end_ms.
- citation objects must be JSON serializable via .to_dict() helper (recommended).

2) EvidenceBundle
A single object that aggregates all evidence available to constrain a response draft.

Required fields (minimal for Batch 0, expandable later):
- action_results: list[ActionResult]
- artifact_results: list[ArtifactResult]
- errors: list[ErrorResult]
- citations: list[Citation]
- notes: list[str]          # freeform, optional

Import the result types from:
- ashby.core.results (Batch 0C)

Rules:
- EvidenceBundle is purely descriptive; it must never execute actions.
- EvidenceBundle is allowed to be empty (chat-only scenario), but judge decisions must reflect that.

3) TruthViolation
A structured report that a response draft violates policy.

Required fields:
- code: str                 # stable identifier, e.g. "FORBIDDEN_PHRASE"
- message: str              # human-readable explanation
- severity: str             # "block" | "warn"
- evidence_required: bool   # whether missing evidence caused the violation

Optional fields:
- meta: dict                # details for debugging/audit

Contract: Policy interface (policy.py)
A TruthPolicy defines *how to evaluate* a draft response given an EvidenceBundle.

Define a Protocol or ABC with canonical method names:

- policy_id: str            # stable identifier
- validate(draft: str, evidence: EvidenceBundle) -> list[TruthViolation]
- rewrite(draft: str, evidence: EvidenceBundle, violations: list[TruthViolation]) -> str | None

Rules:
- validate() must be deterministic.
- rewrite() may return a rewritten string OR None.
- A policy must never claim reality; it only blocks/rewrites language.

Batch 0 MUST include one simple built-in policy to prove the wiring:
- ForbiddenPhrasesPolicy

ForbiddenPhrasesPolicy requirements:
- It MUST replicate the existing legacy phrase list in core/truth_gate.py (same strings).
- It MUST emit TruthViolation(code="FORBIDDEN_PHRASE", severity="block") when triggered.
- Its rewrite() may return the legacy DEFAULT_REJECT string OR None (judge handles blocking).

This policy is not the final truth gate; it is a compatibility bridge.

Contract: Judge (judge.py)
TruthGateJudge is the platform “judge” that applies a TruthPolicy to a draft.

Define these canonical types:

1) TruthGateDecision dataclass
Required fields:
- allowed: bool
- blocked: bool
- rewritten: str | None
- violations: list[TruthViolation]

Rules:
- allowed == True implies blocked == False.
- If rewritten is provided, it must be the string the system should emit.
- If blocked == True and rewritten is None, caller may emit a safe fallback.

2) TruthGateJudge class
Required method:
- evaluate(draft: str, evidence: EvidenceBundle, policy: TruthPolicy) -> TruthGateDecision

Recommended convenience:
- apply(draft: str, evidence: EvidenceBundle, policy: TruthPolicy, fallback: str | None = None) -> str
  Returns rewritten text if present; otherwise returns draft if allowed; otherwise returns fallback or a generic block message.

Rules:
- Judge must not perform NLP or “meaning extraction” beyond calling policy.validate().
- Judge must be deterministic and side-effect-free.
