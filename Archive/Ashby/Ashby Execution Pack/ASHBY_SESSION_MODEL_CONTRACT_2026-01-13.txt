Purpose
Batch 0 adds a minimal “meetings” session container to Ashby’s existing session system.
This is platform work.
It is a placeholder that matches the style of existing sessions (lighting.py, comfort.py) so later router integration for meeting workflows is clean.

Important distinction
This file is NOT the meeting artifact store.
Meeting artifacts live in module-controlled storage (later: Stuart’s artifact tree).
This session container is only for short-lived conversational state (follow-ups, pending confirmations, active meeting selection).

Repo-aware anchor (existing session style)

- <repo_root>/ashby/ashby/brain/sessions/comfort.py

They share these traits:
- module-level dicts keyed by user_id
- timeout-based auto-expiry
- small helper functions (start/get/end)

Batch 0 meetings session MUST follow the same pattern.

Non-goals (explicit)

Contract: meetings session data model (meetings.py)
Define module-level storage:

- _sessions: dict[str, dict]
  Key: user_id (string)

Define a timeout:
- SESSION_TIMEOUT = 60  (seconds)  [default; can be adjusted later]
Rationale:
- This is a conversational follow-up window, not a meeting duration.

Define required functions (canonical names)
1) start_session(user_id: str, *, meeting_id: str | None = None, title: str | None = None, mode: str | None = None, data: dict | None = None) -> dict
Creates/resets a session for the user.
Stores keys:
- "meeting_id": str | None
- "title": str | None
- "mode": str | None
- "data": dict (always a dict; empty if None)
- "updated": float (time.time())

2) end_session(user_id: str) -> None
Deletes session if present.

3) has_active_session(user_id: str) -> bool
Returns True if session exists and is not expired.
If expired: auto-ends and returns False.

4) get_session(user_id: str) -> dict | None
Returns session dict or None (auto-expiring).

5) touch(user_id: str) -> None   (recommended)
If session exists, updates "updated" to now.

Rules:
- The meetings session container must never import any module code.
- It must not read/write disk.
- It must be safe to import in router.py without side effects.
