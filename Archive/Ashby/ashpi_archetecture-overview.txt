AshPi Architecture Overview
===========================

Goal:
-----
AshPi must support multiple brands (Tuya, Nest, NOMA iQ, Neewer, etc.) and multiple interfaces
(Telegram, Discord, web app, voice, Twitch, etc.) without duplicating logic.

Key Design:
-----------
1. Brand-specific modules (per vendor/backend)
   - brands/tuya/*   -> Uses Tuya OpenAPI
   - brands/nest/*   -> Uses Google SDM / Nest APIs
   - brands/noma_iq/*-> Uses Ayla backend (future)
   - brands/neewer/* -> Neewer GL1 / other lights

2. Category separation (by device type)
   - lights (bulbs, panels, strips)
   - plugs (smart outlets, power bars)
   - climate (thermostats, heaters)
   - other: garage doors, fans, cameras, etc.

3. Brand-agnostic core
   - core/devices.py: registry of all devices with:
       - id (cloud/device id)
       - type (bulb/plug/thermostat/etc.)
       - brand (tuya, nest, noma_iq, neewer, etc.)
       - room (bedroom, living_room, office, etc.)
       - groups (wall_lights, ceiling_lights, all_lights, etc.)
   - core logic maps phrases like:
       "wall lights", "room", "all lights", "ceiling"
       into a list of concrete device names.

4. Interfaces (front-ends)
   - interfaces/telegram_bot.py:
       - Reads Telegram messages
       - Parses commands ("make the room brighter")
       - Calls brand/category modules (e.g. tuya.lights.step_brightness)
   - interfaces/web_api.py:
       - HTTP endpoints to control devices from a web UI
   - interfaces/discord_bot.py:
       - Same idea but for Discord
   - future voice/Twitch wrappers do the same.

Principles:
-----------
- No direct Tuya/Nest/NOMA logic inside interface files.
- Interfaces only call into brand/category modules.
- Each brand module knows how to talk to its own backend.
- The core/device registry ties everything together.

Example Flow:
-------------
User (Telegram): "Make the wall lights brighter"

1. Telegram bot parses:
   - target: "wall lights"
   - intent: brighter (increment brightness)

2. core/devices.py:
   - "wall_lights" -> ["captain_america_bulb", "thor_bulb"]
   - brand: tuya
   - category: lights

3. interfaces/telegram_bot.py:
   - calls: brands.tuya.lights.step_brightness(["captain_america_bulb","thor_bulb"], +100)

4. brands/tuya/lights.py:
   - translates that to Tuya OpenAPI calls for those device IDs.
