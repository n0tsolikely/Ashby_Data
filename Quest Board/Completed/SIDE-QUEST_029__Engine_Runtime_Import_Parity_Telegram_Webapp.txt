DESCRIPTION
────────────────────────────────────────────────────────
Running the Ashby/Stuart engine via different entrypoints (Telegram CLI, Webapp via uvicorn)
currently produces inconsistent runtime/import behavior depending on environment configuration.
This includes import failures (e.g., stdlib shadowing such as `secrets.token_hex`) and reliance
on PYTHONPATH or invocation-specific workarounds.

This quest defines and implements a single, clean, canonical runtime/import strategy so the
engine runs consistently across:
- Telegram execution
- Webapp execution
- Future packaging / distribution

────────────────────────────────────────────────────────

────────────────────────────────────────────────────────
