DESCRIPTION
────────────────────────────────────────
Enforce the immutability contract and rerun behavior so canonical artifacts are never silently overwritten. Also lock the v1 media-handling rule: video uploads produce a canonical extracted audio file for processing.

SCOPE / OBJECTIVE
────────────────────────────────────────
Implement and enforce:
- No silent overwrite of canonical artifacts (especially transcripts/formalizations and their manifests).
- Rerun creates a new run_id output path (new run folder) instead of overwriting.
- Session may point to “latest” artifacts via pointers, but old artifacts remain preserved.

Media handling (v1):
- Accept audio and video uploads.
- If input is video: extract audio to a canonical audio file stored within the contribution.
- Keep the original uploaded file when feasible.

This quest does not implement transcription itself; it enforces the storage and rerun semantics that transcription/formalization stages will rely on.

OUT OF SCOPE
────────────────────────────────────────
Implementing diarization/transcription/formalization logic (Dungeon 5).
Cloud storage, remote sync, multi-user auth.
Automatic speaker naming.

DEPENDENCIES
────────────────────────────────────────
- QUEST_012

EXECUTION NOTES
────────────────────────────────────────
Immutability is a trust contract. If this is violated, Stuart becomes un-auditable.
Overlays are separate artifacts; never mutate ground truth to “fix” it.

COMPLETION CRITERIA (REQUIRED)
────────────────────────────────────────
- [x] Two runs created for the same session/contribution do not overwrite each other; each has a unique run_id folder.
- [x] Attempting to write to an existing canonical artifact path fails loudly or routes to a new path (no silent overwrite).
- [x] Session can reference a “latest run” pointer without deleting prior run data.
- [x] Video upload preserves original media and reserves derived_audio_path for later extraction (policy rail locked; extraction staged in later pipeline).
- [x] Contribution manifest reserves fields for original media → derived audio relationship (pointers/paths) without overwriting originals.
