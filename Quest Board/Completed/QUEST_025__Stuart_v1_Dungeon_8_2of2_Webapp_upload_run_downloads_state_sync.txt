DESCRIPTION
────────────────────────────────────────
Complete the primary web door so users can upload media, confirm a plan when required, run the pipeline, and download outputs. Ensure chat overrides UI and UI reflects overridden state.

SCOPE / OBJECTIVE
────────────────────────────────────────
Implement webapp behaviors:
- Upload flow:
  - upload stores contribution (intake)
  - if user message fully specifies plan: proceed
  - otherwise show plan preview + require confirm
- Run execution:
  - trigger pipeline via explicit “go” (confirm button)
  - poll progress and show status
- Downloads:
  - formalization PDF and MD
  - transcript access (may be toggle-gated but must exist)
- State sync:
  - chat commands override UI selection
  - UI updates to reflect overridden mode/speakers/template

This quest is considered done only when the webapp can complete a minimal end-to-end demo.
