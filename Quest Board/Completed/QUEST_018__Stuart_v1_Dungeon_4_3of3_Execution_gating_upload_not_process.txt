DESCRIPTION

Enforce the v1 execution gating law: uploading a file does not automatically trigger heavy processing. Stuart must only run diarize/transcribe/formalize when an explicit “go” signal exists.

SCOPE / OBJECTIVE
────────────────────────────────────────
Implement gating contract from Guild Orders:
- Upload ≠ process.
- Heavy work runs only when explicit “go” exists via:
  - user instruction in chat that fully specifies intent, OR
  - explicit Run/Confirm in webapp, OR
  - Telegram button selections.

Enforce plan preview whenever required input is ambiguous or coming from UI state.
If the user’s message + attachment fully specifies the plan, the system may proceed without extra prompts.
If relying on UI state or ambiguous info, require confirmation.

OUT OF SCOPE

Accounts/auth and advanced permission systems.

EXECUTION NOTES
────────────────────────────────────────
Gating is both privacy and correctness. It also prevents accidental compute burn.
This quest is about enforcing the decision: the system must not “helpfully” process on upload.
