==============================
TALENT_STUART_WEB_DOOR_UPLOAD_RUN_DOWNLOADS — Web Door Upload→Run→Downloads + State Sync + Search
==============================

What it unlocks:
- Upload media in the web UI (plus button) and store it as a contribution (immutable + sha256).
- Plan preview stays mandatory; user explicitly clicks Run before heavy work executes.
- Async run execution with UI polling (no frozen UI).
- Direct downloads for artifacts + exports (formalized.md + formalized.pdf + transcript artifacts).
- Keyword search via /search using SQLite FTS, returning ranked hits with citation anchors.
- UI state sync: when chat/engine resolves defaults/overrides, dropdowns update to match (no divergence).

Files:
- ashby/interfaces/web/app.py
- ashby/interfaces/web/uploads.py
- ashby/interfaces/web/runs.py
- ashby/interfaces/web/static/app.js
- tests/test_meetings_web_upload_run.py

Notes:
- Run: `python scripts/stuart_web.py` (default: http://127.0.0.1:8844)
- Upload contract: POST /api/upload?session_id=... (multipart file)
- Run contract: POST /api/run (async), poll GET /api/runs/{run_id}
- Downloads: GET /download/{run_id}/{filename}
- Search: GET /api/search?q=...&session_id=...
