HANDS BOOT / INSTALL FIELD LOG (SUMMARY)
────────────────────────────────────────
1) Repo placement + venv
- Hands unzipped canon and moved:
  - Ashby_Data
  - Ashby_Engine
  into ~/
- Hands activated an existing venv (ashby-env) and attempted install from Ashby_Engine.

2) Failure class: PyAV / FFmpeg libraries missing (source build)
- During `pip install -r requirements-stuart-v1.txt` pip attempted to build `av` from source.
- The build failed with missing FFmpeg dev libraries:
  - libavformat, libavcodec, libavdevice, libavutil, libavfilter, libswscale, libswresample
  - pkg-config could not find these.

3) Failure class: pip resolver backtracking into older faster-whisper
Root cause (discovered by behavior + dependency graph):
- requirements-stuart-v1.txt constrained `ctranslate2<4.0`.
- faster-whisper 1.0.3 requires `ctranslate2>=4.0`.
- pip responded by downgrading faster-whisper to older releases (0.10.x / 1.0.0 range) that pin older `av==10.*` / `av==11.*`.
- This forced `av` to compile from source (and/or compile incompatible versions), triggering the FFmpeg-dev error.

4) Failure class: pyannote.audio import crash via torchaudio
- pyannote.audio 3.4.0 expects `torchaudio.AudioMetaData`.
- Some newer torchaudio builds do NOT expose AudioMetaData.
- Result was:
  AttributeError: module 'torchaudio' has no attribute 'AudioMetaData'

5) Failure class: NumPy 2 ABI warning + torch interaction
- With newer stacks, torch emitted “module compiled with NumPy 1.x cannot be run in NumPy 2.x”.
- Even when imports succeeded, this is a red flag for stability.

6) Final known-good field resolution achieved by Hands
Hands reached a STRICT-pass preflight by converging on:
- Python 3.10.x
- torch/torchaudio ~= 2.2.2 (CPU)
- numpy < 2 (Hands pinned to 1.26.4)
- openai + deps (sniffio/distro/jiter) installed
- faster-whisper import OK
- pyannote.audio import OK

Result:
- `python scripts/stuart_preflight.py --strict` returned RESULT: OK.

RUNTIME DIRECTORIES CLARIFICATION (OPERATOR CONFUSION RESOLVED)
────────────────────────────────────────
Hands noticed multiple “runtime” looking locations and asked if this was accidental drift:
- `~/ashby_runtime/stuart` (outside the repo)
- `Ashby_Engine/StuartRuntime` (inside the repo)
- `Ashby_Engine/runtime` (inside the repo)

Root cause:
- Stuart’s *meetings* module uses `ashby.modules.meetings.config.get_config()` which defaults STUART_ROOT to:
  `~/ashby_runtime/stuart` (override via `STUART_ROOT`).
- A legacy Telegram control script was defaulting to `Path.cwd() / "StuartRuntime"`, creating an alternate runtime root.
- The repo also contained real run artifacts under `Ashby_Engine/StuartRuntime/`, which can present as “random sessions”
  if any runner points at that directory.

Decision:
- Canon runtime artifacts MUST live outside the repo under `~/ashby_runtime/stuart`.
- In-repo runtime folders should NOT ship with sessions/runs.

Patch applied:
- `io/telegram/ashby_telegram_control.py` now defaults to the shared `get_config().root` runtime.
- Repo-local `Ashby_Engine/StuartRuntime/` was removed from canon (no more baked-in sessions).

────────────────────────────────────────
DECISIONS LOCKED (WHAT WE’RE CHANGING SO THIS DOESN’T HAPPEN AGAIN)
────────────────────────────────────────
1) Requirements hardening
- We will stop allowing pip to “choose” older faster-whisper to satisfy an incompatible ctranslate2 pin.
- We will pin:
  - faster-whisper==1.0.3
  - ctranslate2>=4,<5
  - av>=12.3,<13

2) Torch / torchaudio pinning
- We will explicitly pin torch/torchaudio to a known compatible set for pyannote.audio:
  - torch==2.2.2
  - torchaudio==2.2.2
  (and torchvision==0.17.2 as the matching triple)

3) NumPy pin
- We will pin numpy<2 for Stuart v1 to avoid ABI instability.

4) Preflight improvements
- Preflight will explicitly check and surface the exact failure modes seen in the wild:
  - recommended python window (3.10/3.11)
  - numpy<2 guard
  - torchaudio.AudioMetaData presence guard
  - ctranslate2 minimum version guard
  - av import check
  …with direct fix commands.

5) Documentation update
- INSTALL_STUART_V1.md and SYSTEM_DEPENDENCIES.md must match the real world.
- “one venv” is the default guidance; multiple venvs are only for deliberate isolation.
