3) DECISIONS LOCKED (BINDING)
────────────────────────────────────────────────────────
Decision 1 — Truth Architecture (Product-grade, scalable)
- PLATFORM TruthGateJudge is the single reusable judge.
- Modules provide domain TruthPolicy implementations (MeetingsTruthPolicy for Stuart).
- Module-local “truth rules” are restricted to internal data validation invariants only.
  If the rule is about reality (decisions, actions, attendance, factual assertions), it belongs in:
  platform judge + module policy.

Decision 2 — Execution Profiles / Consent
- ONE execution profile substrate across the platform.
- No parallel profile systems per module.
- Clarifier: execution profiles are NOT personality/tone.

Decision 3 — Routing + Removability
- Core router remains thin.
- Module routers handle module intent.
- Removing a module must not leave dead module-specific logic in core.

Decision 4 — File Discipline
- Big files are OK when cohesive.
- God files are banned.
- Prefer separated capabilities that call each other; keep together only when separation is worse.

Decision 5 — Test Discipline
- “Tests passed” claims require real execution in the environment.
- No “reasoning tests” or “looks fine” substitutes.

Decision 6 — Canon Export Rule (Process)
- Every change to Ashby_Data or Ashby_Engine must ship as an updated combined zip.
- Each delivered zip is treated as the new canonical state going forward.
