4) WHAT EACH NEW ENGINE PRIMITIVE DOES (REAL FUNCTIONALITY, NOT HYPE)

4A) StorageAdapter interface (ashby/interfaces/storage.py)
- validate_rel_path(rel_path): normalizes and rejects unsafe paths:
  - rejects absolute paths (leading “/”)
  - rejects “..” traversal
  - converts backslashes to forward slashes for safety/determinism
- StorageAdapter Protocol defines required methods:
  - root(), exists(), mkdir(), list(), read_bytes(), write_bytes()
- StorageAdapterMixin provides convenience:
  - read_text/write_text, read_json/write_json
- write_bytes contract:
  - overwrite=False must refuse overwrite (FileExistsError)
  - atomic=True must write temp file + fsync + os.replace

4B) Storage paths helpers (ashby/storage/paths.py)
- ensure_dir(Path) helper
- storage_root(base_dir), plus:
  - profiles_dir(root)
  - sessions_dir(root)
  - results_dir(root)
- sanitize_slug(name, max_len=48): deterministic safe filenames/ids
- safe_relpath(*parts): safe join + normalize using validate_rel_path

4C) Local filesystem storage implementation (ashby/storage/local_fs.py)
- LocalFSStorageAdapter implements StorageAdapter on disk.
- Guarantees:
  - rooted namespace
  - path traversal blocked
  - atomic writes by default (mkstemp in same dir + fsync + os.replace)
  - refuses overwrite unless overwrite=True
- Includes best-effort directory fsync for durability.

4D) Results (ashby/core/results.py)
- Canonical results objects:
  - ActionResult (world-touching actions like lights.set)
  - ArtifactResult (durable artifacts like transcripts)
  - ErrorResult (structured failures)
- Shared base fields:
  - ok, kind, started_at, ended_at, correlation_id, errors
- ErrorInfo includes structured (code, message, detail).
- Helpers:
  - ok_action(...)
  - ok_artifact(...)
  - err(...)
- Round-trip support:
  - to_dict() and from_dict()

4E) Profile + Consent gate (ashby/core/profile.py)
- ExecutionProfile enum:
  - LOCAL_ONLY, HYBRID, CLOUD
- EgressPlan describes intended network egress:
  - provider, endpoint, data_categories, purpose, retention
- ConsentRecord models explicit user consent
- evaluate_profile(profile, egress, consent) returns ProfileGateDecision:
  - LOCAL_ONLY: denies any egress
  - HYBRID: requires consent matching provider + categories superset
  - CLOUD: allows egress
  - No egress planned: always allowed

4F) Truth Gate spine (ashby/core/truth/*)
- evidence.py
  - Citation (points into durable artifacts)
  - EvidenceBundle aggregates:
    - action_results, artifact_results, errors, citations, notes
  - TruthViolation structured policy violations
- policy.py
  - TruthPolicy Protocol
  - ForbiddenPhrasesPolicy bridges to legacy ashby.core.truth_gate phrase list/reject behavior (does NOT refactor legacy file)
- judge.py
  - TruthGateJudge:
    - evaluate(draft, evidence, policy) returns TruthGateDecision
    - apply(draft, evidence, policy) returns rewritten, original, or fallback
- Key note:
  - Current spine policy implemented is the compatibility policy (forbidden phrases).
  - Additional policies like “action-claim must be backed by evidence” are NOT implemented yet; those come when we wire the gate into runtime and define those rules.

4G) Module Registry (ashby/core/module_registry.py)
- ModuleSpec: metadata + factory
- ModuleRegistry: in-memory registry:
  - register(), get(), list_ids(), create()
- Intended use: Stuart registers here cleanly; future modules (lighting, inventory, etc.) also register here.
