HANDS FIELD BOOT LOG (WHAT HAPPENED ON THE MACHINE)
────────────────────────────────────────
Context: Hands had two venvs (ashby-env on Python 3.12 and stuart-env_py310 on Python 3.10) and wanted a clean, repeatable boot sequence.

Key events / errors observed:
1) `pip install -r requirements-stuart-v1.txt` in the Python 3.12 env tried to build PyAV from source and failed due to missing FFmpeg dev libraries (pkg-config could not find libavformat/libavcodec/etc). This was a symptom of the resolver selecting a source build.

2) In the Python 3.10 env, pip still backtracked into older `faster-whisper` versions that required `av==10.*` / `av==11.*`, triggering a build from source and Cython/noexcept compilation failures. Root cause: the repo requirement pin for `ctranslate2<4` forced pip to avoid `faster-whisper 1.0.x` (which requires ctranslate2>=4).

3) `pyannote.audio` import failed with:
   `AttributeError: module 'torchaudio' has no attribute 'AudioMetaData'`
   This happened after installing newer torchaudio builds (2.10.0+cpu) that no longer exposed that attribute used by `pyannote.audio 3.4.0`.

4) Torch + NumPy ABI mismatch warnings appeared when running some newer Torch builds alongside NumPy 2.x.

5) Preflight STRICT fail occurred due to `openai` import failing when dependencies were skipped (installed `openai` with `--no-deps`), specifically missing `sniffio`.

Hands’ final working resolution (strict preflight OK):
- Use Python 3.10
- Pin NumPy to `<2` (resolved ABI warnings path)
- Use torch/torchaudio versions that include `AudioMetaData` (2.2.2 worked)
- Ensure OpenAI deps are installed (sniffio/distro/jiter)

Result:
- `python scripts/stuart_preflight.py --strict` returned `RESULT: OK`.
