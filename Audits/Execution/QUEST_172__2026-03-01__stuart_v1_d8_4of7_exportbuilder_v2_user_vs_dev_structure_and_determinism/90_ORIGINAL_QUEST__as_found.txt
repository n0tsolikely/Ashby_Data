## EXECUTIVE SUMMARY
Dungeon 8 requires a user-facing export zip structure and a dev export variant — both deterministic and traceable — while avoiding zombie export plumbing.

The repo already has an export implementation (`ashby/modules/meetings/export/bundle.py`), but it exports the **internal** storage layout and always includes raw JSON + dev artifacts.

This quest replaces/extends export builder behavior to produce:
- USER_EXPORT: user-facing bundle (no raw JSON outputs)
- DEV_EXPORT: same user-facing layout + a `dev/` folder with receipts/evidence/raw JSON

## HARD REQUIREMENTS (DUNGEON 8)
### Bundle classes
- USER_EXPORT:
  - include user-facing artifacts only
  - exclude raw JSON outputs (minutes.json, transcript_version.json, evidence_map, llm usage receipt, run manifests)

- DEV_EXPORT:
  - includes everything in USER_EXPORT
  - plus `dev/` folder containing:
    - evidence_map.json
    - llm_usage_receipt.json
    - run manifests/logs
    - raw JSON outputs

### Zip structure
- `session.json` (session_id + session_title)
- `audio/`
- `transcripts/<transcript_version_id>/`
- `formalizations/<formalization_id>/`
- `overlays/` (speaker_map overlays; active overlay included)
- `dev/` (DEV_EXPORT only)

### Determinism
- No absolute paths inside zip.
- Stable ordering.
- No recursive inclusion of previous exports.
- Deterministic bytes where feasible (avoid writing export-time timestamps into content).

## IMPLEMENTATION PLAN
### A) Create a v2 builder (recommended)
- Create a new module:
  - `ashby/modules/meetings/export/bundle_v2.py`
- Keep `bundle.py` only if still used anywhere. Otherwise delete or convert to wrapper calling v2.

### B) Deterministic zip writer helper
- Implement a small helper class (e.g., `ZipDeterministicWriter`) that can:
  - add bytes with stable ZipInfo timestamps
  - add files by reading bytes and using file mtime
  - enforce sorted iteration

### C) USER_EXPORT content generation
1) `session.json`
- Generate minimal JSON:
  - `session_id`
  - `session_title` (title_override if present else session_id)

2) `audio/`
- Include all contribution files tied to the session (deterministic naming).
- Suggested naming inside zip:
  - `audio/<contribution_id>__<original_filename>`

3) `transcripts/<transcript_version_id>/`
- Transcript versions are stored as JSON segments. USER_EXPORT needs text artifacts.
- For each transcript_version:
  - generate `transcript.txt` if `txt` requested
  - generate `transcript.md` if `md` requested
  - generate `transcript.pdf` if `pdf` requested

Formatting rules:
- Use speaker overlay mapping for that transcript_version_id (SessionState v2 mapping).
- Speaker display:
  - if mapped: show mapped name only
  - if not mapped: show `Speaker-01`, `Speaker-02`, ... (UI 1-based)

4) `formalizations/<formalization_id>/`
- Include formalization outputs for each run that produced a formalization:
  - `minutes.md/pdf` and/or `journal.md/pdf` based on requested `formalization_formats`
- Exclude minutes.json/journal.json in USER_EXPORT.

5) `overlays/`
- Include at least:
  - active overlay mapping(s) needed to reproduce the included transcript text.

Suggested structure:
- `overlays/transcripts/<transcript_version_id>/speaker_map.json`

### D) DEV_EXPORT additions
- Add `dev/` containing:
  - `dev/formalizations/<formalization_id>/evidence_map.json`
  - `dev/formalizations/<formalization_id>/llm_usage_receipt.json`
  - `dev/formalizations/<formalization_id>/run.json`
  - `dev/formalizations/<formalization_id>/events.jsonl`
  - `dev/transcripts/<transcript_version_id>/transcript_version.json`
  - plus raw minutes.json/journal.json if present

### E) Traceability metadata inside exported artifacts
Dungeon 8 demands ID traceability.

Implement in export-generated artifacts (recommended):
- MD: add an HTML comment header (invisible to rendered markdown) like:
  - `<!-- session_id: ...; transcript_version_id: ...; formalization_id: ...; generated_at: ... -->`
- TXT: add a small visible header block (first lines) with IDs.
- PDF:
  - transcripts: if you generate via reportlab, add a small footer line with IDs.
  - formalizations: update the PDF render path to include a footer (see QUEST_173 if you prefer separating).

**generated_at MUST be stable** (use artifact created_ts or run created_ts), not export-time.
