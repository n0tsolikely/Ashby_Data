  - transcript_version_id must map deterministically to the run and the exact artifact.

This quest locks the ID strategy and implements the internal service layer that enumerates
transcript versions for a session by scanning run artifacts (bounded, deterministic).

    ------------------------------------------------
    DESCRIPTION
    ------------------------------------------------
    Define transcript_version_id strategy and implement a service that can:
- list transcript versions for a session
- load a transcript version into a canonical JSON payload with segments

This is the substrate for:
- /api/sessions/{id}/transcripts
- /api/transcripts/{transcript_version_id}

We do NOT fabricate segments. Ever. We only read transcript artifacts produced by the pipeline.


    ------------------------------------------------
    SCOPE / OBJECTIVE
    ------------------------------------------------
    A) Decide transcript_version_id strategy (must be consistent with Quest 090 contract doc)

   Recommended v1:
     transcript_version_id = run_id   (simple, stable, already unique)
   OR
     transcript_version_id = "tv__" + run_id

   Hard rule:
     - Must be reversible: tv_id -> run_id without database lookups.
     - Must not collide with other IDs (run_id vs transcript_id).

B) Implement transcript version derivation service (new module)

   Create one canonical module, e.g.:
     - ashby/modules/meetings/transcripts/versions.py
     OR
     - ashby/modules/meetings/transcript_versions.py

   Service functions (suggested):
     - list_transcript_versions(session_id: str, limit:int, offset:int) -> list[TranscriptVersionSummary]
     - get_transcript_version(transcript_version_id: str) -> TranscriptVersion (full)
     - helpers:
         - _run_has_transcript(run_dir) -> bool
         - _select_best_transcript_artifact(run_dir) -> Path
           Preference order:
             1) aligned_transcript.json (best; includes speakers if diarized)
             2) transcript.json (fallback)
         - diarization_enabled inferred:
             - True if aligned_transcript.json exists OR any segment includes speaker labels
             - False otherwise

   TranscriptVersionSummaryV1 fields (minimum):
     - transcript_version_id
     - session_id
     - run_id
     - created_ts (from run.json)
     - diarization_enabled (bool)
     - asr_engine ("default" for now)
     - segment_count
     - duration_ms (optional if available)

   TranscriptVersionV1 fields (minimum):
     - transcript_version_id
     - session_id
     - run_id
     - diarization_enabled
     - segments: [{segment_id, start_ms, end_ms, speaker_label?, text, confidence?}]
     - source_artifact: {kind, filename, sha256?}
     - evidence anchors:
         - segment_id must be stable per version
         - include timestamps

C) Tests
   - Add unit tests using fixtures:
       - tests/fixtures/stub_transcript_segments.json
     or by creating a fake run dir under tmp STUART_ROOT with transcript.json and run.json.
   - Assert:
       - aligned_transcript preferred
       - diarization_enabled inference correct
       - deterministic list ordering

D) Docs update
   - Update contract doc to declare transcript_version_id strategy.
