DESCRIPTION
────────────────────────────────────────────────────────
The session transcripts list endpoint currently scans runs and returns full transcript JSON blobs. That’s not production-grade and it’s not real transcript versioning.

This quest switches the list endpoint to use the TranscriptVersion runtime index and returns metadata only (fast), with pagination.

────────────────────────────────────────────────────────
SCOPE / OBJECTIVE
────────────────────────────────────────────────────────
Update /api/sessions/{session_id}/transcripts:

- Source of truth becomes:
  - STUART_ROOT/sessions/{session_id}/transcripts/index.jsonl

- Response items must include at least:
  - transcript_version_id
  - session_id
  - run_id
  - created_ts
  - diarization_enabled
  - asr_engine
  - segments_count
  - active: bool (true iff equals session_state.active_transcript_version_id)

- MUST NOT include full transcript segments in list response (metadata-only).
- Add stable sorting:
  - primary: created_ts desc
  - secondary: transcript_version_id lexicographic desc

- Keep existing response envelope shape:
  { "session_id": ..., "transcripts": [...], "page": {...} }

Update api_models_v1 TranscriptVersionSummaryV1 accordingly.

────────────────────────────────────────────────────────
OUT OF SCOPE
────────────────────────────────────────────────────────

- UI changes (later quests will adapt).
