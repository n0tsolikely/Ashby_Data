SCOPE
1) Session state schema evolution:
   - Add a v2-compatible structure to store overlay pointers per transcript version, e.g.:
     - speaker_overlays_by_transcript: { "<transcript_version_id>": "<overlay_id or null>" }
   - Keep legacy field active_speaker_overlay_id for compatibility, but define it as:
     - active_speaker_overlay_id := speaker_overlays_by_transcript[active_transcript_version_id] (if present)
2) Backward compatible API surface:
   - set_active_speaker_overlay(session_id, overlay_id):
     - MUST update the overlay pointer for the ACTIVE transcript_version_id of that session.
     - MUST also keep legacy active_speaker_overlay_id consistent.
   - Add explicit helpers:
     - get_speaker_overlay_for_transcript(session_id, transcript_version_id)
     - set_speaker_overlay_for_transcript(session_id, transcript_version_id, overlay_id_or_none)
3) Seed-forward behavior:
   - When a new transcript_version_id is created in a session, if that transcript has no overlay pointer:
     - Copy/seed the most recent ACTIVE overlay pointer from the previous active transcript_version_id.
     - “Seed” can be a pointer reference (overlay artifacts are immutable), not a file copy.
4) Migration behavior (must be deterministic):
   - When reading an old session_state file that has active_speaker_overlay_id but no map:
     - Interpret it as mapping for active_transcript_version_id.
     - When the state is next written, persist the v2 structure.
