SCOPE
Backend — add transcript-scoped speaker map contract:
1) Update GET /api/transcripts/{transcript_version_id}
   - Include these fields in the response:
     - speaker_map: { "SPEAKER_00": "Greg", ... } (empty dict if none)
     - speaker_overlay_id: "<overlay_id>" | null
     - speakers: ["SPEAKER_00", "SPEAKER_01", ...] (unique labels derived from transcript segments)
2) Add GET /api/transcripts/{transcript_version_id}/speaker_map
   - Returns: { "speaker_overlay_id": ..., "speaker_map": {...} }
3) Add PUT (or POST) /api/transcripts/{transcript_version_id}/speaker_map
   Request body:
     - mapping: dict[str,str]  (may be empty)
     - author: optional str ("user" default)
   Behavior:
     - If mapping is empty OR all values are empty/whitespace:
       - Clear the speaker overlay pointer for that transcript_version_id.
       - Return speaker_map={} and speaker_overlay_id=null.
     - Else:
       - Create a new overlay artifact (immutable).
       - Set overlay pointer for transcript_version_id (QUEST_154 structure).
       - Return the saved mapping + overlay id.
   Side effect:
     - Refresh sqlite speaker_maps rows for (session_id, transcript_version.run_id) to reflect the new ACTIVE mapping.
       - If mapping cleared: delete rows for that (session_id, run_id).
       - If mapping saved: delete then insert new rows.

Backend — keep compatibility and avoid zombie endpoints:
4) Update POST /api/runs/{run_id}/speaker_map
   - Resolve run_id → transcript_version_id (if possible).
   - Delegate to the transcript endpoint logic so it updates transcript-scoped overlay pointers, not session-global.
   - Keep response shape compatible for the UI, but mark via log/comment that transcript-scoped is canonical.
