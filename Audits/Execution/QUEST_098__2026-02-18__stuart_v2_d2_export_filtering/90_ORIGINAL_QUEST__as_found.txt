    ------------------------------------------------
    DESCRIPTION
    ------------------------------------------------
    Implement export_type filtering logic so we can create deterministic zip bundles:
  - full_bundle: everything relevant to the session (receipts + artifacts)
  - transcript_only: transcript substrates + receipts (no formalization outputs)
  - formalization_only: formalization outputs + evidence + receipts (no raw transcript unless required as evidence)

Determinism is mandatory:
  - stable sort order
  - stable timestamps (fixed)
  - stable permissions


    ------------------------------------------------
    SCOPE / OBJECTIVE
    ------------------------------------------------
    A) Extend export module API

   In ashby/modules/meetings/export/bundle.py:
     - Add export_type parameter to export_session_bundle:
         export_session_bundle(session_id, export_type="full_bundle", ...)

     - Add helper that classifies files into categories:
         - receipts/manifests (session.json, run.json, contribution.json, session_state.json, overlays)
         - transcripts (transcript.json, aligned_transcript.json, diarization.json, segment maps)
         - formalizations (output.json, output.md, output.pdf, evidence_map.json, exports/)
         - run logs (events.jsonl, inputs/resolved_input.json)

   Define exact inclusion rules (v1):
     full_bundle:
       - include everything currently included (except exports/ bundles recursion guard remains)
     transcript_only:
       - include session.json + session_state.json (pointer)
       - include overlays (speaker map) (optional but recommended)
       - include contribution source files + receipts (because transcript alone without source is less useful)
       - include run.json + events.jsonl for runs that produced transcripts
       - include transcript artifacts only (exclude exports pdf/md and formalization outputs)
     formalization_only:
       - include session.json + session_state.json
       - include overlays (speaker map) (recommended)
       - include run.json + events.jsonl for runs that produced formalizations
       - include formalization artifacts: output.* + evidence maps + exports
       - include resolved_input.json receipts (to prove which contribution it came from)
       - (Optional) include aligned_transcript.json if evidence_map requires it; define the rule explicitly.

   HARD RULE:
     - No bundle should include a prior bundle zip (exports/ recursion guard).

B) Add deterministic unit tests
   - New test file suggested:
     Ashby_Engine/tests/test_meetings_export_bundle_export_type_filters.py

   Tests:
     - Create fake session structure under tmp STUART_ROOT with:
         - session.json
         - one contribution with source
         - one run with transcript artifacts
         - one run with formalization outputs + primary_outputs
     - Assert:
         - transcript_only bundle includes transcript artifacts and excludes output.pdf/md/json
         - formalization_only includes output artifacts and excludes transcript artifacts (unless rule says otherwise)
         - full_bundle includes both sets
     - Assert deterministic zip (same file list ordering; timestamp fixed)

C) Docs update
   - Ensure Quest 090 contract doc has the final inclusion rules.
