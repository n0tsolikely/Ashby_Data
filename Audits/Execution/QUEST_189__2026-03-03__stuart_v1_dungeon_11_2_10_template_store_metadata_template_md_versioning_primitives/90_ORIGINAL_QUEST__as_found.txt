SYSTEM CONTEXT STATEMENT
- Dungeon 11 requires templates to be first-class artifacts:
  - Stable `template_id`
  - Human-facing `template_title`
  - Versioned (immutable prior versions)
  - Stored under TemplateStoreRoot (runtime), not Ashby_Data and not in-repo
- We already have a **TemplateSpec parser** in `template_registry.py` for system templates, but no runtime-backed store for user templates.

REPO / MONSTER ANCHORS
- Runtime root + layout (TemplateStoreRoot introduced in QUEST_188):
  - ashby/modules/meetings/layout.py (StuartLayout.templates)
  - ashby/modules/meetings/init_root.py (creates TemplateStoreRoot)
- Template parsing (will be reused, not duplicated):
  - ashby/modules/meetings/template_registry.py (parse_markdown_template)

CONSTRAINT SUMMARY
- Storage contract (user templates):
  - `<STUART_ROOT>/templates/user/<mode>/<template_id>/v<version>/metadata.json`
  - `<STUART_ROOT>/templates/user/<mode>/<template_id>/v<version>/template.md`
- **No in-place edits**: every change creates a new `v<version>` directory.
- Keep a single seam for the root (layout/templates).
- Do not invent a database for templates yet; filesystem store only.

ANTI-DUP PLAN
- `rg -n "TemplateStore|template_store|templates/user" Ashby_Engine/ashby`
- If any TemplateStore-like module exists already, extend it. If not, create exactly one new module.

PLACEMENT INTENT

- Tests:

TASK BREAKDOWN
1) Create module skeleton
   - Add: `Ashby_Engine/ashby/modules/meetings/templates/store.py`
   - Export functions from `ashby/modules/meetings/templates/__init__.py` (create if missing)

2) Define the on-disk layout helpers
   - `template_root(layout) -> Path`
   - `template_dir(mode, template_id) -> Path`
   - `template_version_dir(mode, template_id, version:int) -> Path`

3) Implement TemplateRecord
   Required fields (Dungeon 11):
   - template_id: str  (stable)
   - template_title: str (renamable, but renames create a new version)
   - version: int
   - mode: Literal["meeting","journal"]
   - template_text: str  (markdown)
   - defaults: {
       include_citations: bool,
       show_empty_sections: bool
     }
   - created_at: ISO 8601 str
   - updated_at: ISO 8601 str

4) Implement create_template (new template)
   - Inputs: mode, title, template_text, defaults
   - Output: TemplateRecord (version=1)
   - Writes:
     - metadata.json (all fields except template_text)
     - template.md (template_text)

5) Implement create_new_version (edit existing template)
   - Inputs: template_id, mode, template_text/title/defaults updates
   - Computes next version: max(existing)+1
   - Writes new version folder; never edits old versions

6) Implement list_templates
   - Returns latest version per template_id for a mode
   - Include `source: "user"` in returned descriptors (system handled later)

7) Implement delete_template
   - Deletes the entire template_id directory for that mode (all versions)
   - Requires explicit caller confirmation parameter at API layer (handled in later quest)

8) Unit tests
   - Add: `tests/test_template_store_filesystem_versioning.py`
   - Cover:
     - create_template -> v1 exists
     - create_new_version -> v2 exists and v1 unchanged
     - list_templates returns latest only
     - load_template returns correct version
     - delete_template removes tree
