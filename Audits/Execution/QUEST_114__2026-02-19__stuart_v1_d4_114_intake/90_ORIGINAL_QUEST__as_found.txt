────────────────────────────────────────────────────────
DESCRIPTION
────────────────────────────────────────────────────────
Introduce the canonical TranscriptVersion artifact so a single session can have multiple transcripts (different engines, diarization toggles, retries) and we can choose which transcript feeds formalization.

This is the foundational storage + schema quest. It must be immutable (write-once) and future-proof for additional transcript engines and fields.

────────────────────────────────────────────────────────
SCOPE / OBJECTIVE
────────────────────────────────────────────────────────
Implement a TranscriptVersion artifact schema (v1) and storage API that supports:
- transcript_version_id (use prefix `trv_` via ids.new_id("trv"))
- session_id
- run_id (the run that produced the transcript)
- created_ts (float seconds)
- diarization_enabled: bool
- asr_engine: str (start with "default")
- segments: list[{segment_id:int, start_ms:int, end_ms:int, speaker?:str, text:str, confidence?:float}]
- audio_ref metadata sufficient for traceability (MUST NOT be absolute paths outside STUART_ROOT; store safe relpaths)

Runtime storage (under STUART_ROOT):
A) Per-session storage (source of truth)
- sessions/{session_id}/transcripts/
  - index.jsonl (append-only rows: transcript_version_id, run_id, created_ts, diarization_enabled, asr_engine, segments_count, sha256, etc.)
  - versions/{transcript_version_id}.json  (write-once; full artifact)

B) Global lookup for fast resolution (required for /api/transcripts/{id})
- transcript_versions/lookup.jsonl
  - append-only rows: transcript_version_id, session_id, run_id, created_ts

Provide a python API in ashby/modules/meetings/ (e.g. transcript_versions.py) with:
- create_transcript_version(...)  (writes versions/{id}.json, appends both index.jsonl + lookup.jsonl)
- list_transcript_versions(session_id) -> summaries (no segments)
- load_transcript_version(session_id, transcript_version_id) -> full
- resolve_transcript_version(transcript_version_id) -> {session_id, run_id?} (uses lookup.jsonl)
- ensure_transcripts_dirs(session_id)

All writes must be atomic + deterministic JSON formatting.
