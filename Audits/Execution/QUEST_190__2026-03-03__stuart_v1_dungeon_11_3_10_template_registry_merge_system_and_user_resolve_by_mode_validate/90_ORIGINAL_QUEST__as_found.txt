SYSTEM CONTEXT STATEMENT
- Stuart currently supports **system templates** only via:
  - `ashby/modules/meetings/template_registry.py`:
    - `_SYSTEM_DIR` points at in-repo templates
    - `_V1_TEMPLATES_BY_MODE` is hard-coded to default templates
- Dungeon 11 requires **user templates** to be discoverable and selectable:
  - Listed per mode
  - Validated per mode
  - Resolved by template_id + version
  - Titles displayed in UI via registry payload
- We must not fork parsing logic: reuse existing parse_markdown_template + defaults behavior.
