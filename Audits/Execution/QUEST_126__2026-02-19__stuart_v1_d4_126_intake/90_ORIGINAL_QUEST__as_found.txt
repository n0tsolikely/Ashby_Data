DESCRIPTION
────────────────────────────────────────────────────────
We already have runs on disk with transcript.json/aligned_transcript.json but no TranscriptVersion artifacts. We must support them without forcing a manual migration step.

This quest adds a safe back-compat layer.

────────────────────────────────────────────────────────
SCOPE / OBJECTIVE
────────────────────────────────────────────────────────
Implement one (or both) of the following in a safe way:

Option A (lazy migration on access):
- When listing transcripts for a session:
  - If index.jsonl missing or empty, scan the session’s runs for transcript artifacts
  - For each discovered transcript artifact, create a TranscriptVersion artifact (write-once) and append to index
  - Set active_transcript_version_id to the newest discovered transcript version if not already set

Option B (virtualization without writing):
- If index missing, synthesize transcript versions from run artifacts in-memory with deterministic ids:
  - transcript_version_id = tv__{run_id} (legacy)
  - Mark these as legacy-only in response

Pick Option A if you want to converge to the new storage model automatically.

Important: Must NOT break existing tv__run_id URLs.

────────────────────────────────────────────────────────
OUT OF SCOPE

- One-time CLI migration tooling (optional).
- Deleting legacy artifacts.
